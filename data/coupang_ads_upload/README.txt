쿠팡 광고 CSV/Excel 업로드 폴더
================================

[권장 다운로드 파일 타입]
✓ pa_daily_vendorItem_YYYYMMDD_YYYYMMDD.xlsx  (일별 × 상품 — 권장!)
✗ pa_total_campaign_YYYYMMDD_YYYYMMDD.xlsx    (캠페인 기간 합계 — 날짜 없음, 사용 불가)

반드시 "일별 (daily)" 로 나뉜 파일을 받아야 대시보드에서 7일/14일/30일
필터가 정상 동작합니다.

[다운로드 경로]
1. 쿠팡 Wing → 광고센터 → 광고보고서
2. 맞춤 보고서 또는 광고 보고서 → 기간 구분 "일별" 선택
3. 다운로드 (xlsx / csv)

[자동 누적 병합 동작 방식]
매주 30일치 CSV 를 업로드하면, 이전에 저장된 데이터와 병합됩니다:
  - 새 파일의 날짜 범위에 해당하는 기존 데이터는 "최신 값" 으로 교체
  - 그 외 날짜의 기존 데이터는 보존 (히스토리 유지)

예시:
  주차 1 (2026-04-23): 03-24 ~ 04-22 (30일) 업로드 → 기본 저장
  주차 2 (2026-04-30): 03-31 ~ 04-29 (30일) 업로드
    → 03-31 ~ 04-22 기존 데이터가 새 값으로 교체 (수치 보정 반영)
    → 03-24 ~ 03-30 원래 데이터 그대로 유지
    → 04-23 ~ 04-29 신규 추가
  주차 3 (2026-05-07): 04-07 ~ 05-06 (30일)
    → 마찬가지 로직으로 04-30 ~ 05-06 추가되면서 기존 보존

시간이 지날수록 대시보드에는 90일/180일 전 데이터까지 누적 보유됩니다.

[파일 드롭 → 실행]
1. 이 폴더에 새 파일 드롭 (파일명 자유, 기존 파일 그대로 두거나 삭제 OK)
2. 자동 실행: sync_all.bat 이 매일 오전 10시 최신 파일 자동 처리
3. 수동 실행: .venv\Scripts\python.exe scripts\sync_coupang_ads_csv.py

[캠페인 → 브랜드 자동 분류]
- "김똑똑" "똑똑떡뻥" "떡뻥" "똑똑연구소" "로켓그로스" → 쿠팡_똑똑연구소
- "AI 광고" "AI광고" "롤라루" "캐리어" "백팩" → 쿠팡_롤라루
- 기타 → 쿠팡 (미분류 — utils/products.py 의 COUPANG_AD_BRAND_KEYWORDS 수정)

[지원 포맷]
- .xlsx / .xls (엑셀 — openpyxl 로 처리)
- .csv (UTF-8-sig / cp949 / euc-kr 자동 탐지)

[필수 컬럼 (이름 유연 매칭)]
- 날짜 (YYYYMMDD 정수, YYYY-MM-DD, YYYY/MM/DD 모두 지원)
- 캠페인명
- 광고비 / 비용
- (선택) 노출수, 클릭수
- (선택) 총 주문수(14일), 총 전환매출액(14일) — 14일 어트리뷰션 우선

[주의]
- 개인정보 없음 (캠페인명 + 수치만)
- 원본 xlsx 는 .gitignore 로 public repo 에서 제외 (파싱된 parquet 만 커밋)
- 처리된 파일도 폴더에 계속 유지 (재처리/디버그 용)
- 오래된 파일은 수동 삭제 권장
