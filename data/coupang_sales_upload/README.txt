쿠팡 Wing 상품별 판매 CSV 업로드 폴더
========================================

[왜 이 폴더가 필요한가]
쿠팡 로켓배송(벤더 풀필먼트) 주문은 판매자 Open API 로 조회 불가 →
Wing 에서 매출 리포트 CSV 수동 다운로드 → 여기 드롭 → 자동 파싱해서
orders.csv 에 누적 병합.

판매자 판매 / 로켓그로스 주문은 기존 쿠팡 Wing API 로 자동 수집되므로
이 폴더에는 주로 "로켓배송 상품별 판매" 리포트만 업로드하면 됨.

[다운로드 경로 — 쿠팡 Wing]
1. https://wing.coupang.com/ 로그인
2. 좌측 메뉴: 판매 관리 또는 통계 → 매출 리포트 / 상품별 판매 리포트
3. 기간 선택 (최근 30일 권장)
4. 기간 구분 반드시 "일별" 선택 (날짜 컬럼 필수)
5. CSV / Excel 다운로드 → 이 폴더에 드롭

[필요한 컬럼 (이름 유연 매칭)]
필수:
- 날짜 / 일자 / 집계일 (YYYYMMDD 또는 YYYY-MM-DD 형태)
- 상품명 / 제품명 / productName
- 매출 / 판매금액 / 실결제금액 / 정산금액

선택:
- 판매수량 / 수량 (없으면 1 로 계산)
- 주문번호 (없으면 자동 해시 생성)
- 배송방식 (로켓배송 / 로켓그로스 / 업체배송 구분용)

[자동 브랜드 분류]
상품명 키워드 기반:
- "김똑똑", "떡뻥", "똑똑연구소" → store='쿠팡_똑똑연구소'
- "롤라루", "캐리어", "백팩", "여행" → store='쿠팡_롤라루'
- 매칭 실패 → store='쿠팡' (미분류)

[누적 병합 동작 — 매주 30일 CSV 업로드 패턴]
새 파일의 날짜 범위에 해당하는 기존 쿠팡 데이터만 교체,
그 외 날짜의 기존 데이터는 보존.

예시:
  주차 1 (04-23): 03-24 ~ 04-22 (30일) → 저장
  주차 2 (04-30): 03-31 ~ 04-29 (30일)
    → 03-31~04-22 교체 / 03-24~03-30 보존 / 04-23~04-29 추가
  점진적으로 90일+ 히스토리 축적

[실행]
수동:   .venv\Scripts\python.exe scripts\sync_coupang_sales_csv.py
자동:   sync_all.bat 에 포함됨 (매일 오전 10시)

[주의]
- 개인정보 없음 (상품명 + 수량 + 금액만)
- 원본 CSV 는 .gitignore 로 public repo 에서 제외
- 파싱 결과만 orders.csv → Streamlit Cloud 반영
