쿠팡 Supplier Hub 발주리스트 CSV 업로드 폴더
==============================================

[개념 정리 — 중요!]
쿠팡 로켓배송(벤더 풀필먼트)은 쿠팡이 상품을 매입 → 쿠팡이 소비자 판매.
벤더가 볼 수 있는 건:
  ✓ 쿠팡으로 발주/입고된 수량 (벤더 → 쿠팡 B2B)
  ✗ 쿠팡 → 소비자 최종 판매 (비공개)

따라서 이 폴더의 CSV 는 "소비자 판매" 가 아니라 "벤더 발주 B2B" 데이터입니다.

[대시보드 반영 방식]
- 매출 분석 페이지: 구글 시트 매출만 사용 (발주 데이터 미포함)
- 제품 분석 페이지: 제품별 × 일별 수량/금액에 발주 데이터 포함
  → 판매 채널: "쿠팡 벤더 발주" 로 표시 (실 판매와 시각적 분리)
  → store 값: 쿠팡_롤라루_벤더 / 쿠팡_똑똑연구소_벤더
- 저장 파일: data/coupang_inbound.csv (orders.csv 와 분리)

[다운로드 경로 — 쿠팡 Supplier Hub]
1. https://supplier.coupang.com/ 로그인
2. 물류 → 발주리스트 (또는 애널리틱스 → 일간 종합 성과 지표)
3. 기간: 최근 30일 권장
4. 엑셀/CSV 다운로드 (여러 파일 가능)
5. 이 폴더에 전부 드롭 (파일 개수 제한 없음 — 스크립트가 자동 순회)

[지원 포맷]
- .xlsx / .xls (엑셀)
- .csv (UTF-8 / cp949 / euc-kr 자동 탐지)

[필수 컬럼 (이름 유연 매칭)]
- 일자 / 날짜 / 일시 / 발주일
- 상품명 / 제품명 / 첫 SKU명
- 수량 / 발주수량 / 판매수량
- 금액 / 매출 / 발주금액 / 실공급가

[자동 처리]
1. 오즈키즈 제품 등 타 브랜드 자동 차단 (utils/products.py PRODUCT_BLOCKLIST_KEYWORDS)
2. 상품명 키워드로 브랜드 분류:
   - "김똑똑/떡뻥/똑똑연구소" → 쿠팡_똑똑연구소_벤더
   - "롤라루/캐리어/백팩/여행" → 쿠팡_롤라루_벤더
   - 기타 → 쿠팡_벤더_기타
3. 25개 파일 모두 순차 파싱 → concat → 중복 제거 (date × store × product 합산)
4. coupang_inbound.csv 에 누적 병합 (과거 데이터 보존)

[누적 병합 — 매주 업로드 패턴]
새 CSV 의 날짜 범위만 교체, 그 외 날짜의 기존 데이터 보존.
예시:
  주차 1: 03-24 ~ 04-22 업로드 → 30일 저장
  주차 2: 03-31 ~ 04-29 업로드
    → 03-24 ~ 03-30 보존
    → 03-31 ~ 04-22 교체
    → 04-23 ~ 04-29 추가

[실행]
수동: .venv\Scripts\python.exe scripts\sync_coupang_sales_csv.py
자동: sync_all.bat 에 포함 (매일 오전 10시)

[주의]
- 발주리스트 ≠ 소비자 판매
- 매출 분석에는 반영 안 됨 (구글 시트 기준)
- 제품 분석에서만 "쿠팡 벤더 발주" 채널로 표시
- 원본 CSV 는 .gitignore 로 public repo 제외, 파싱 결과(csv)만 커밋
