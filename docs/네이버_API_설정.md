# 네이버 검색광고 API 설정 가이드

네이버 검색광고 데이터를 매일 자동으로 대시보드에 반영하려면 이 단계를 따르세요. 한 번 설정하면 이후 손댈 필요 없습니다.

---

## 1단계 · API 사용 신청

1. **네이버 검색광고 시스템** 로그인 (https://searchad.naver.com)
2. 우측 상단 **「도구」** 메뉴 → **「API 사용 관리」** 클릭
3. **「API 사용 신청」** 버튼
   - 서비스명: `그로잉업팀 대시보드`
   - 용도: `자사 광고 성과 모니터링 및 분석`
4. 약관 동의 후 신청 완료
5. **승인 소요: 보통 당일~1영업일** (가끔 즉시 승인되기도 함)
6. 승인 알림 메일을 받으면 다음 단계로

---

## 2단계 · 키 3가지 확인

승인 후 **API 사용 관리** 페이지에 접속하면:

### 2-1. API Key (액세스라이선스)
- "액세스라이선스" 섹션의 첫 번째 문자열
- 예시 형태: `010000000000xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
- 공개돼도 큰 위험은 아니지만 개인 정보 취급

### 2-2. Secret Key (비밀키)
- 처음 생성 시 **한 번만 전체 문자열이 표시됨** — 반드시 복사해 안전한 곳에 보관
- 잊어버리면 재생성 필요 (기존 키 무효화됨)
- 예시 형태: `AQAAAAA+2MdFL9N1AbcDefGhiJkLmNoPqRsTuVwXyZ==`
- **외부 노출 시 즉시 재발급** 권장

### 2-3. Customer ID (광고주 ID)
- 네이버 검색광고 시스템 상단의 광고주 계정 번호
- 로그인 후 좌측 상단 프로필 영역에서 확인
- 형태: 7~9자리 숫자 (예: `1234567`)

---

## 3단계 · 대시보드에 등록

1. 대시보드 좌측 사이드바 → **「API 연결」** 페이지
2. **네이버 검색광고 API** 섹션의 **「키 입력 / 변경」** 펼치기
3. 3개 값 입력:
   - API Key
   - Secret Key
   - Customer ID
4. **"이 기기에 저장"** 체크 (권장) → 프로젝트 루트에 `.env` 파일로 저장됨
5. **「저장 및 연결 테스트」** 클릭

성공하면 `"연결 성공. 전체 캠페인 N개 (활성 M개)."` 메시지가 뜹니다.

---

## 4단계 · 첫 동기화

1. 같은 페이지의 **「데이터 동기화」** 섹션
2. "며칠 전부터" = 30 (최근 30일 데이터 당겨오기)
3. "기준일" = 어제
4. **「지금 동기화」** 클릭 → 약 30초~1분 대기 (일자당 약 2초)
5. 미리보기에서 ROAS·전환수 확인 후, 자동으로 병합됨
6. **홈** 페이지로 이동하면 실제 네이버 데이터가 반영된 대시보드 확인

## 5단계 · 매일 자동 갱신 (선택)

Windows 작업 스케줄러로 매일 아침 자동 실행:

### 작업 등록
1. `Win + R` → `taskschd.msc` Enter
2. 우측 **「기본 작업 만들기」**
3. 이름: `그로잉업팀_네이버동기화`
4. 트리거: **매일** / **오전 6:30** (네이버 확정 데이터 반영 이후)
5. 동작: **프로그램 시작**
6. 프로그램/스크립트:
   ```
   C:\Users\PC\ddokddok-dashboard\.venv\Scripts\python.exe
   ```
7. 인수 추가 (옵션):
   ```
   C:\Users\PC\ddokddok-dashboard\scripts\sync_naver_ads.py --days 3
   ```
8. 시작 위치 (옵션):
   ```
   C:\Users\PC\ddokddok-dashboard
   ```
9. **작업 우클릭 → 속성 → 일반 탭**:
   - "사용자가 로그온했는지에 관계없이 실행" 선택
   - "가장 높은 수준의 권한으로 실행" 체크

### 테스트
작업 목록에서 **우클릭 → 실행**으로 즉시 실행.
`data/sync_log.txt` 파일에서 결과 확인.

## 자주 발생하는 이슈

| 증상 | 원인 | 해결 |
|------|------|------|
| 연결 테스트 401/403 | API Key/Secret 오타 또는 만료 | API 사용 관리 페이지에서 재확인, 필요 시 Secret 재발급 |
| Customer ID 찾기 어려움 | 광고주 계정 번호와 네이버 일반 ID 혼동 | 검색광고 시스템 상단 "광고주 ID" (숫자)만 사용 |
| 당일 데이터가 너무 적음 | 네이버 집계 지연 (15분~1시간) | 전일까지만 동기화, 당일은 다음날 아침에 반영 |
| 전환매출액이 0 | 네이버 전환 추적 스크립트 미설치 또는 미매핑 | 검색광고 → 도구 → 전환추적 설정 확인 |
| 캠페인 0개 | 신규 계정 또는 다른 Customer ID | 검색광고 시스템 상단의 광고주 ID 재확인 |
| 스케줄러 실행 안 됨 | Python 경로 오류 또는 .env 누락 | `scripts/sync_naver_ads.py`를 직접 실행해 에러 확인 |

## "실시간"에 대한 솔직한 설명

- 네이버 검색광고 API는 **배치 기반**이며, 초/분 단위 진짜 실시간이 아닙니다.
- 통계 데이터는 **15분~1시간 지연**으로 제공됩니다.
- **당일 데이터는 계속 갱신되며 최종값이 아닙니다.**
- 정확한 확정치는 **다음날 오전**에 반영됩니다.
- 대시보드 의사결정 용도로는 **매일 아침 자동 갱신이면 충분**합니다.
- 시간당 갱신이 꼭 필요하면 작업 스케줄러에 매시간 트리거를 추가할 수 있습니다 (드문 요구사항).

---

## 보안 권장사항

- `.env` 파일은 **절대 공유하거나 스크린샷에 노출하지 마세요**
- 프로젝트 폴더 전체를 클라우드 동기화 (OneDrive, Google Drive) 시 `.env`는 제외하세요
- Secret Key가 노출된 정황이 있으면 즉시 **API 사용 관리 → 비밀키 재발급**
- 공용 PC에서는 "이 기기에 저장" 체크 해제하고 세션 단위로만 사용
