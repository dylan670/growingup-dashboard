# 네이버 데이터 연결 가이드

대시보드에 실제 네이버 데이터를 넣는 3단계입니다. 한 번 설정하면 이후는 CSV 드래그만으로 갱신됩니다.

---

## 연결해야 할 소스 2개

| 소스 | 제공 데이터 | 사용 위치 |
|------|------------|-----------|
| **네이버 검색광고** (searchad.naver.com) | 노출/클릭/광고비/전환/매출 (일자별) | 채널별 ROAS, 소재 피로도, 예산 증감 판단 |
| **네이버 스마트스토어** (sell.smartstore.naver.com) | 주문 내역 (주문자, 상품, 금액) | 재구매 트래커, 고객 세그먼트, 실 매출 집계 |

**중요:** 두 시스템이 분리되어 있어 각각 CSV를 받아와야 합니다. 검색광고의 "전환매출"은 네이버 픽셀이 집계한 수치라 실제 스마트스토어 매출과 차이가 있습니다. 대시보드는 광고비·전환 평가엔 검색광고 CSV를, 실제 매출·재구매엔 스마트스토어 CSV를 씁니다.

---

## 1단계 · 네이버 검색광고 CSV 다운로드

1. **네이버 검색광고 시스템** 로그인 (https://searchad.naver.com)
2. 상단 메뉴 **「보고서」** → **「다차원 보고서」** 또는 **「성과 리포트」**
3. 리포트 설정:
   - **조회 기간**: 최근 30일 또는 90일 (길수록 대시보드 트렌드가 정확)
   - **차원**: **일자** (필수). 키워드·그룹까지 쪼개면 자동 합산되므로 그대로 둬도 OK
   - **지표**: 노출수 / 클릭수 / 총비용 / 전환수 / 전환매출액 (최소 이 5개)
4. **「다운로드」** 버튼 → CSV 또는 엑셀 선택 (엑셀이면 저장 시 `CSV UTF-8`으로 변환)

**받은 파일 예시 컬럼:** `일자, 캠페인, 광고그룹, 키워드, 노출수, 클릭수, 총비용, 전환수, 전환매출액`

→ 대시보드의 **데이터 설정** 페이지 → **"네이버 검색광고 CSV"** 업로드

---

## 2단계 · 네이버 스마트스토어 주문 CSV 다운로드

1. **스마트스토어 판매자센터** 로그인 (https://sell.smartstore.naver.com)
2. 좌측 메뉴 **「주문/배송」** → **「주문 내역 조회」**
3. 조회 조건:
   - **조회 기간**: 최근 60일 (네이버 정책상 한 번에 3개월까지 조회 가능)
   - **주문 상태**: 전체 또는 "결제완료" 이상
4. 조회 후 상단 **「엑셀 다운로드」** 또는 **「대량 엑셀 다운로드」** 클릭
5. 다운로드된 `.xlsx` 파일을 엑셀에서 열어 **"CSV UTF-8"** 로 저장
   - (또는 그대로 `.xlsx` 업로드도 지원 예정 — 현재는 CSV만)

**받은 파일 예시 컬럼:** `상품주문번호, 주문번호, 결제일, 구매자ID, 구매자명, 상품명, 수량, 상품별 총 주문금액, ...` (40여 개)

→ 대시보드의 **데이터 설정** 페이지 → **"네이버 스마트스토어 주문 CSV"** 업로드

---

## 3단계 · 대시보드에서 업로드

1. 대시보드 좌측 사이드바에서 **「데이터 설정」** 클릭
2. 페이지 중간 **"네이버 원본 CSV 업로드 (자동 변환)"** 섹션
3. 두 파일 각각 업로드 → **「네이버 데이터 병합」** 클릭
4. 자동 수행되는 작업:
   - CSV 인코딩 감지 (UTF-8 / CP949 / EUC-KR)
   - 한글 컬럼명 → 대시보드 표준 컬럼으로 매핑
   - 키워드별 데이터가 있으면 일자별로 합산
   - 구매자ID / 이름 / 연락처 → 익명 해시 ID로 변환 (개인정보 보호)
   - 기존 `data/ads.csv`·`data/orders.csv`의 "네이버" 행만 교체 (쿠팡/자사몰은 유지)
5. 홈 화면으로 이동하면 네이버 수치가 실제 데이터로 반영됨

## 자주 발생하는 이슈

| 증상 | 원인 | 해결 |
|------|------|------|
| "인코딩 해석 실패" | 엑셀에서 일반 저장 → ANSI 인코딩 | 엑셀 → "다른 이름으로 저장" → **CSV UTF-8** 선택 |
| "필수 컬럼을 찾을 수 없음" | 리포트 설정에서 지표 누락 | 검색광고 리포트에서 "노출수/클릭수/총비용/전환수/전환매출액" 모두 체크 |
| 재구매율 0% | 스마트스토어 CSV에 "구매자ID" 누락 | 다운로드 옵션에서 '주문자 정보' 포함 체크 후 재다운로드 |
| 날짜가 안 맞음 | "결제일"과 "주문일"이 다른 달에 걸침 | 조회 기간을 "결제일" 기준으로 설정 |
| 매출 0 | "전환매출액" 대신 "클릭 가치" 등 다른 지표 | 리포트 지표에 **전환매출액** 명시적으로 추가 |

## 업데이트 주기 권장

- **일일 운영**: 매일 아침 검색광고 CSV만 다운로드 업로드 (5분)
- **주간 리뷰**: 월요일에 주문 CSV까지 포함해 재구매 트래커 확인 (10분)
- **월간 리뷰**: 월초에 전월 데이터로 CRM 세그먼트 / 예산 재배분 결정

## 다음 단계: API 자동 연동 (로드맵)

### 네이버 검색광고 API
- 공식 문서: https://naver.github.io/searchad-apidoc/
- 인증: API Key + Secret Key + Customer ID (HMAC-SHA256 서명)
- 신청 경로: 검색광고 시스템 → 도구 → API 사용 관리 → **「API 사용 신청」**
- 승인 소요: 보통 1~2 영업일
- 승인 후 대시보드에 키를 입력하면 매일 자동 갱신 가능

### 네이버 커머스 API (스마트스토어 주문 자동 조회)
- 공식 문서: https://apicenter.commerce.naver.com/
- 인증: OAuth 2.0 (client_credentials)
- 신청 경로: 커머스 API 센터 → **「파트너 신청」**
- 승인 소요: 3~7 영업일
- 승인 후 주문 데이터 실시간 수집 가능

**현재 MVP는 CSV 수동 업로드**로 충분히 작동하므로, API는 운영이 안정된 후 추가해도 됩니다.
