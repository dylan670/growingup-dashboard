# 그로잉업팀 마케팅 대시보드

## 어떤 대시보드인가요?

**그로잉업팀(OPENHAN)** 이 운영하는 3개 브랜드의 매출·광고·재구매·제품을 한 화면에서 통합 관리하는 의사결정 대시보드입니다.

원래는 매출 분석만 가볍게 보려고 만들기 시작했는데, 만들다 보니 광고 효율, 재구매 알림, 캠페인 진단까지 **팀이 매일 쓰는 대부분의 운영 데이터를 한 곳에 모으는 방향**으로 확장됐습니다.

운영 중인 브랜드:
- **똑똑연구소** — 유아식 (김똑똑 어린이김, 똑똑떡뻥)
- **롤라루** — 여행용품 (캐리어, 백팩)
- **루티니스트** — 신규 (다이어리, 러닝 액세서리)

## 기술 스택

**Streamlit** 으로 작업 중이며 GitHub를 통해 **Streamlit Community Cloud**로 배포하는 구조입니다. 회사 PC가 매일 10시에 자동으로 5개 API에서 데이터를 받아 git에 push하면 클라우드가 자동 재배포합니다.

데이터를 끌고오는 방식은 **API 자동 수집 + 구글시트 + 수동 CSV 업로드** 세 가지를 섞어 씁니다. 어드민 자체 API가 없거나(쿠팡 로켓배송 벤더 발주, 쿠팡 광고센터) Open API가 없는 채널(무신사·오프라인·이지웰·오늘의집)은 수동 시트 입력 + 수동 CSV 업로드로 보완합니다.

업로드된 데이터와 자동 수집 데이터는 모두 **CSV + Parquet**으로 저장합니다 (별도 DB 안 씀).

LLM(Claude/GPT) **호출은 현재 0건**이고, 모든 인사이트와 알림은 임계값/수식 기반의 룰 엔진으로 만듭니다 (AI 환각 위험 0%, 매번 같은 입력엔 같은 결과).

```
매출(쿠팡 로켓배송): 쿠팡 Supplier Hub CSV 수동 업로드 (벤더 발주 — Open API 없음)
매출(무신사·오프라인·이지웰·오늘의집) : 구글 시트 수동 입력 (Open API 없음)

광고(자사몰 메타) : Meta Marketing API 자동 수집 (3개 광고 계정)

광고(쿠팡)       : 쿠팡 광고센터 CSV 수동 업로드 (Open API 없음)

제품·고객·재구매 : 위 API 주문 데이터에서 자동 추출
인사이트·예측·알림: 자체 룰 엔진 + 가중 통계 (LLM 미사용)
```

## 자동 갱신 구조

```
매일 오전 10:00 (회사 PC)
  └─ 5개 API에서 7일치 재수집 (정산 지연/주문 수정/취소 자동 흡수)
  └─ 시트 live 다운로드 → Parquet 저장
  └─ 룰 엔진 인사이트/알림 사전 계산 → JSON 저장
  └─ git commit + push
       ↓
GitHub repo 업데이트 (자동 트리거)
       ↓
Streamlit Cloud 자동 재배포 (3~5분)
       ↓
팀원 브라우저에서 새로고침하면 최신 데이터
```

원하는 시점에 강제로 갱신하고 싶으면 사이드바 **"🔄 전체 API + 시트 갱신"** 버튼 한 번. (회사 PC에선 진짜 sync 트리거, 클라우드에선 캐시만 클리어)

## 시트 vs API 데이터 정합성 정책

시트는 **공식 매출 수치**로 신뢰하고, API는 보조 신호로 쓰는 정책입니다.
- 시트에 입력 있으면 → 시트값 표시
- 시트가 비어 있으면 → API 자동 수집값으로 보완 (📌 ⚡ 마커)
- 시트값이 API보다 훨씬 작으면 → "누락 의심" 으로 자동 표시 (운영팀이 시트 보강하라는 신호)

이 통합 로직 덕에 매출 분석 / 홈 대시보드 / 인사이트 / 월말 예측 모든 페이지가 정확히 같은 수치를 씁니다.

## 목적

- 그로잉업팀 3브랜드(똑똑연구소·롤라루·루티니스트)의 다중 채널 매출·광고·재구매를 한눈에 파악하기 위함.
- 매일 10시 자동 갱신되는 데이터를 보고 그날의 운영 액션(광고비 증감 / 시트 보강 / CRM 발송 / 캠페인 정지)을 정함.

## 핵심 목표

1. **다중 채널 매출 통합 모니터링** — 자사몰 / 네이버 스스 / 쿠팡(로켓그로스 + 로켓배송 벤더 + 판매자판매) / 무신사 / 오프라인 / 이지웰 / 오늘의집 까지 8개 이상 채널 합산 매출 + 목표 달성률
2. **광고 효율 자동 진단** — 캠페인별 ROAS / CTR 피로도 / 증액·축소·낭비 의심 신호 자동 분류
3. **재구매 타이밍 관리** — 30~60일 경과 고객 자동 추출 → CRM 발송용 CSV 다운로드
4. **제품 단위 성과 분석** — 채널 별로 흩어진 SKU 정규화 + 캠페인-제품 가중 상관 (어떤 광고가 어떤 제품 견인하는가)
5. **월말 매출 예측** — 가중 통계법(EWMA × 요일 계절성 × 추세 보정)으로 산술평균보다 정확한 페이스 예측

## 한계 / 수동 처리 부분

- **무신사 / 오프라인 / 이지웰 / 오늘의집** Open API가 없어서 운영팀이 매일 시트에 수동 매출 입력. 일별 표에는 반영되지만 제품 단위 분석은 불가.
- **쿠팡 로켓배송 벤더 발주** Wing API로 못 잡혀서 쿠팡 Supplier Hub에서 CSV 다운받아 사이드바 "📤 CSV 업로드" 페이지에 올리는 방식.
- **쿠팡 광고** 광고주용 Open API가 없어서 광고센터 리포트 CSV를 다운받아 같은 페이지로 업로드. 일별 리포트(`pa_daily_*`)와 합계 리포트(`pa_total_campaign_*`) 둘 다 지원.

## 팀 다른 프로그램과의 관계

팀원들이 각자 만들고 있는 분석 도구들과 **실행 속도와 사용 빈도를 보고 통합/분리** 하는 방향으로 운영합니다. 이 대시보드는 "매일 아침 10분 안에 봐야 하는 핵심 KPI + 액션 신호"에 집중하고, 디테일 분석(프로모션 캘린더 디자인, 재고 상세 등)은 별도 도구나 시트에서 보는 방식이 효율적일 것 같다고 봅니다.
