# 그로잉업팀 마케팅 대시보드

**그로잉업팀**이 운영하는 유아동 식품 스토어 2개 (**똑똑연구소** · **롤라루**)의 통합 마케팅 의사결정 대시보드.
네이버 스마트스토어 · 쿠팡 로켓그로스 · 자사몰(메타 광고) 3개 채널을 한 화면에서 관리하고,
**광고비 증/감, 소재 교체, CRM 리마인더** 같은 액션 신호를 자동 감지합니다.

**운영 스토어:**
- 똑똑연구소 — 김똑똑 어린이김, 100% 유기농 쌀과자 똑똑떡뻥
- 롤라루

## 구성 (MVP)

- **홈** — 3가지 핵심 질문에 대한 즉답
  1. 이번 기간 가장 효율 좋은 채널은?
  2. 쿠팡 신규 리뷰 1건당 매출 기여는?
  3. 재구매 주기 도달 고객, 리마인더 보냈나?
- **채널별 심화** — 네이버/쿠팡/자사몰 각각의 ROAS 트렌드, CTR 피로도, 예산 증감 시뮬레이터
- **알림 센터** — 6개 룰 기반 자동 감지 (증액/축소/소재 피로/재배분/재구매/주문 감소)
- **재구매 트래커** — 재구매 주기 도달 고객 리스트 + CRM 대상 CSV 다운로드
- **데이터 설정** — CSV 업로드, 샘플 데이터 재생성

## 빠른 시작 (Windows)

### 1. 사전 준비

- **Python 3.10 이상** 설치: https://www.python.org/downloads/
- 설치 시 ⚠️ **"Add Python to PATH"** 체크박스 반드시 선택

### 2. 실행

프로젝트 폴더의 `run.bat`을 더블클릭하세요.

- **첫 실행**: 가상환경 생성 + 패키지 설치 (약 3~5분)
- **이후**: 바로 실행 (10초 이내)

실행되면 브라우저가 자동으로 열리며 `http://localhost:8501`에서 대시보드가 표시됩니다.

### 3. 수동 실행 (선택)

```bash
cd C:\Users\PC\ddokddok-dashboard
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

## 데이터 연결

첫 실행 시 `data/` 폴더에 60일치 **샘플 데이터**가 자동 생성됩니다. 샘플에는 의도적으로 "자사몰 CTR 피로도", "쿠팡 증액 기회", "재구매 대상 고객" 같은 시나리오가 심겨 있어 대시보드 동작을 확인할 수 있습니다.

### 실제 데이터로 교체

1. 각 플랫폼에서 CSV 다운로드
   - **네이버**: 판매자센터 → 정산관리 / 통계
   - **쿠팡**: 윙 → 매출관리 / 광고관리
   - **메타**: 광고 관리자 → 내보내기
   - **자사몰**: 카페24/아임웹 등 주문 내역
2. 대시보드의 **데이터 설정** 페이지에서 업로드
   - 또는 `data/ads.csv`, `data/orders.csv`, `data/reviews.csv`를 직접 교체

### CSV 스키마

**`data/ads.csv`** (광고 성과)
```
date, channel, spend, impressions, clicks, conversions, revenue
```

**`data/orders.csv`** (주문 내역, 재구매 분석용)
```
date, order_id, customer_id, channel, product, quantity, revenue
```

**`data/reviews.csv`** (선택, 리뷰 기여도 분석용)
```
date, channel, product, rating, text
```

- `channel` 값은 **네이버**, **쿠팡**, **자사몰** 중 하나 (정확히 일치해야 집계됨)
- `date`는 `YYYY-MM-DD` 형식

## 커스터마이징

### 목표 ROAS 조정

`utils/metrics.py` 상단의 `TARGET_ROAS` 값을 실제 사업 수치로 수정:

```python
TARGET_ROAS = {
    "네이버": 4.0,   # 400%
    "쿠팡": 5.0,     # 500%
    "자사몰": 2.5,   # 250%
}
```

### 알림 룰 임계값 조정

`utils/alerts.py`에서 각 함수의 조건식 수정:

- 증액 신호를 더 보수적으로: `target * 1.3` → `target * 1.5`
- 소재 피로도 기준 완화: `0.30` → `0.20`
- 채널 재배분 감도: `50` → `30`

### 재구매 주기 변경

**재구매 트래커** 페이지의 슬라이더로 14~60일 사이 조정 가능. 기본값 30일.
